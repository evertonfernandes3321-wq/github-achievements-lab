# Contribuindo

Este repositorio e um laboratorio de boas praticas GitHub. Contribuicoes devem ser pequenas, claras e seguras.

## Padrao de branch

Use nomes curtos e descritivos:

- `docs/descricao-curta`
- `chore/descricao-curta`
- `ci/descricao-curta`

## Padrao de commit

Prefira mensagens no estilo Conventional Commits:

- `docs: melhora readme`
- `chore: ajusta configuracao`
- `ci: adiciona verificacao simples`

## Padrao de pull request

Cada pull request deve explicar:

- o que mudou;
- por que mudou;
- quais arquivos foram afetados;
- se ha risco de publicar dados sensiveis.

## Checklist antes do merge

- [ ] O diff foi revisado.
- [ ] Nenhum arquivo `.env` foi incluido.
- [ ] Nenhuma chave privada, token ou credencial foi incluida.
- [ ] A mudanca tem escopo pequeno.
- [ ] O titulo do PR descreve a alteracao.
