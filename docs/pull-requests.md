# Pull Request Checklist

Use this checklist before opening or merging a pull request in this lab.

## Before opening

- Start from the latest `main`.
- Keep the branch focused on one change.
- Review the diff locally.
- Confirm that no secret-like file was added.
- Write a title that describes the change.

## Pull request body

Include:

- a short summary;
- the files or areas changed;
- any safety checks performed;
- whether the change is documentation, CI or maintenance.

## Before merging

- Confirm the GitHub Actions checks passed when present.
- Confirm the branch is mergeable.
- Prefer squash merge for one logical change.
- Delete the branch after merge.
