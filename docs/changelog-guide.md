# Changelog Guide

The changelog should summarize meaningful changes without duplicating every commit.

## When to update

Update `CHANGELOG.md` when a pull request changes:

- repository structure;
- required documentation;
- workflow behavior;
- contribution or review guidance.

## Entry format

Use a dated section and short bullets:

```text
## 0.3.0 - YYYY-MM-DD

- Adds a documentation check workflow.
- Adds a pull request review guide.
```

## Writing style

- Use present tense.
- Keep bullets short.
- Group related changes in the same version.
- Do not include secrets, private links or internal production details.
