# Branching Guide

Branches in this lab should be short lived and easy to connect to a pull request.

## Naming pattern

Use one of these prefixes:

- `docs/` for documentation changes.
- `ci/` for workflow changes.
- `chore/` for repository maintenance.

## Examples

- `docs/add-review-guide`
- `ci/update-docs-check`
- `chore/update-topics`

## Rules of thumb

- Start from the latest `main`.
- Keep one logical change per branch.
- Delete merged branches after the pull request is merged.
