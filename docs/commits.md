# Commit Message Guide

Commit messages should explain the intent of a change without requiring the reader to inspect every file first.

## Format

Use a short Conventional Commits style prefix:

```text
type: short description
```

## Common types

- `docs`: documentation only.
- `ci`: workflow or automation changes.
- `chore`: maintenance without user-facing behavior.

## Examples

- `docs: adiciona guia de branches`
- `ci: atualiza verificação de documentação`
- `chore: organiza tópicos do repositório`

## Good commit habits

- Keep commits focused on one logical change.
- Avoid mixing unrelated documentation and workflow edits.
- Include co-author trailers only for real collaboration.
