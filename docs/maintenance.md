# Maintenance Guide

This lab should stay small, readable and safe to use for GitHub workflow practice.

## Weekly review

- Check open pull requests and close stale work when it is no longer useful.
- Confirm the default branch is clean.
- Review whether documentation links still point to existing files.
- Check that GitHub Actions runs are passing.

## Repository hygiene

- Keep documentation files focused.
- Avoid adding generated files unless they are part of the example.
- Delete merged branches.
- Keep topics aligned with the repository purpose.

## When to stop changing the repo

Stop adding changes when a new pull request does not improve documentation, workflow safety or maintainability.
