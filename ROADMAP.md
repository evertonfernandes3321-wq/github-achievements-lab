# Roadmap

Este roadmap organiza melhorias simples para manter o laboratorio util e seguro.

## Proximos passos

- Criar templates leves para issues e pull requests.
- Adicionar uma verificacao simples de Markdown em CI.
- Documentar exemplos de nomes de branch e commits.
- Registrar decisoes de fluxo quando o repositorio evoluir.

## Ideias de melhorias

- Checklist automatizado para arquivos sensiveis comuns.
- Guia curto para revisar pull requests documentais.
- Exemplos de changelog para pequenas entregas.
- Secao de perguntas frequentes sobre fluxo GitHub.

## Boas praticas

- Preferir PRs pequenos e objetivos.
- Evitar commits grandes com assuntos misturados.
- Manter a `main` sempre em estado publicavel.
- Nunca publicar tokens, chaves privadas ou arquivos de ambiente.
- Usar squash merge quando fizer sentido para preservar historico limpo.
